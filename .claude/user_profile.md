The challenge has three rounds — Round 1 (data/scoring, this repo), Round 2 (case study), Round 3 (deck + presentation to Amex decision scientists). The Round 1 method must be both leaderboard-competitive AND presentable/defensible to a senior data scientist.

User is engaged with the economics and the "why" behind decisions — asks conceptual questions (e.g., what does zero-floor mean, what does raw weight mean) and wants to deeply understand, not just run code.

**How to apply:** Frame explanations with economic intuition first, then the math. Don't just state what the code does — explain why the decision is correct from a card P&L perspective.
